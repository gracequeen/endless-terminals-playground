# Checkpoint Resume Fix Summary

## Issues Found

1. **Buffer not flushing**: save_checkpoint() didn't use context manager and didn't flush/fsync
   - Result: Checkpoint file might not be written to disk before crash

2. **Semantic mismatch**: Saved batch_num as 'last_line' but used it to skip individual lines
   - Example: If 5 batches processed (5000 records), checkpoint saved 5, resume only skipped 5 lines
   - Result: Reprocessed almost everything from the beginning

3. **Data overwriting**: Used INSERT OR REPLACE instead of accumulating
   - Result: On resume, previously aggregated data was replaced instead of accumulated

4. **No batch commits**: Only committed at the end
   - Result: Data loss if crash occurred

## Fixes Applied

1. **save_checkpoint()**: Added context manager, flush(), and os.fsync()
2. **Line tracking**: Changed to track actual line numbers (i+1) instead of batch numbers
3. **SQL**: Changed to INSERT ON CONFLICT DO UPDATE to accumulate values
4. **Commits**: Added conn.commit() after each batch
