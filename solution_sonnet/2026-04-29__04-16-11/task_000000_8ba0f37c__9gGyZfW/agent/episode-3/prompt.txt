WHY SSH WORKS BUT HTTP DOESN'T:
- SSH (port 22): Has a simple unconditional ACCEPT rule with no string matching requirements
- HTTP/HTTPS (ports 80/443): Require the string "Mozilla" to be present in the packet data
- When curl connects without a custom User-Agent, it sends "curl/X.X.X" which doesn't contain "Mozilla"
- The traffic fails the string match, bypasses the ACCEPT rule, and hits the subsequent DROP rules
