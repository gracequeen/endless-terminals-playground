> # Timezone Bug Fix Summary
>
> ## Problem
> The scheduler app was sending meeting reminders at incorrect times for meetings between 2am-3am local time (Europe/Berlin), specifically during DST transition
s. The issue only occurred when run via cron, causing a 1-hour offset.
>
> ## Root Cause
> In `app.py`, the `convert_to_berlin()` function had a critical bug:
>
> **OLD (BUGGY) CODE:**
> ```python
> def convert_to_berlin(utc_time_str):
>     utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
>     berlin_dt = BERLIN_TZ.localize(utc_dt)  # BUG: Treats UTC time as Berlin time!
>     return berlin_dt
> ```
>
> This incorrectly treated the parsed UTC datetime as if it were already in Berlin timezone.
>
> ## Solution
> **NEW (FIXED) CODE:**
> ```python
> def convert_to_berlin(utc_time_str):
>     utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
>     utc_dt_aware = UTC_TZ.localize(utc_dt)      # First: localize as UTC
>     berlin_dt = utc_dt_aware.astimezone(BERLIN_TZ)  # Then: convert to Berlin
>     return berlin_dt
> ```
>
> This properly:
> 1. Localizes the naive datetime as UTC
> 2. Converts it to Berlin timezone using `astimezone()`
> 3. Correctly handles DST transitions
>

> March 31, 2024 is when Europe/Berlin transitions from CET (UTC+1) to CEST (UTC+2) at 2am local time.
