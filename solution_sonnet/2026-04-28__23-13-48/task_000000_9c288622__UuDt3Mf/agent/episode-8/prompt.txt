> # Bug Fix Report: Timezone Conversion Issue
>
> ## Problem
> The scheduler application was sending meeting reminders at incorrect times, with errors of exactly 1 hour for meetings between 2am-3am local time (Europe/Berl
in).
>
> ## Root Cause
> In `app.py`, the `convert_to_berlin()` function had a critical bug:
>
> **BUGGY CODE:**
> ```python
> def convert_to_berlin(utc_time_str):
>     utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
>     berlin_dt = BERLIN_TZ.localize(utc_dt)  # WRONG!
>     return berlin_dt
> ```
>
> The bug: `BERLIN_TZ.localize(utc_dt)` treats the naive datetime as if it's already in Berlin time, rather than treating it as UTC time that needs conversion.
>
> **FIXED CODE:**
> ```python
> def convert_to_berlin(utc_time_str):
>     utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
>     utc_dt = UTC_TZ.localize(utc_dt)        # First: make it UTC-aware
>     berlin_dt = utc_dt.astimezone(BERLIN_TZ)  # Then: convert to Berlin
>     return berlin_dt
> ```
>
> ## Impact
> - All meetings were affected (1-2 hour differences depending on DST)
> - The "Maintenance Window Prep" meeting at 2024-03-31T01:30:00 UTC should be at 03:30 CEST (after DST spring forward), but was being calculated as 01:30 CET
> - This caused reminders to be sent at the wrong time
>
> ## Why Manual Testing Didn't Catch It
> The `test_conversion.py` script worked correctly because it used `datetime.now(berlin)` which properly handles timezone-aware datetimes. The bug only manifest
