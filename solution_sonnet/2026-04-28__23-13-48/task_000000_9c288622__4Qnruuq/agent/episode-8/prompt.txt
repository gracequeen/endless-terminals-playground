# Scheduler DST Bug Fix Summary

## Problem
Meeting reminders were being sent at incorrect times - off by exactly one hour, but **only for meetings between 2am-3am local time** (Europe/Berlin). This speci
fically affected the Monday 2:30am maintenance window reminder.

## Root Cause
The bug was in the `convert_to_berlin()` function in `/home/user/scheduler/app.py`.

### The Buggy Code:
```python
def convert_to_berlin(utc_time_str):
    utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
    # BUG: This treats the UTC time as if it's already Berlin time
    berlin_dt = BERLIN_TZ.localize(utc_dt)
    return berlin_dt
```

**What was wrong:**
- `BERLIN_TZ.localize()` was called on a naive datetime parsed from a UTC string
- This incorrectly treated the UTC timestamp as if it were already in Berlin timezone
- During DST transitions (when clocks jump from 2am to 3am), this caused a 1-hour offset

### The Fix:
```python
def convert_to_berlin(utc_time_str):
    utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
    # FIX: First localize as UTC, then convert to Berlin
    utc_dt_aware = UTC_TZ.localize(utc_dt)
    berlin_dt = utc_dt_aware.astimezone(BERLIN_TZ)
    return berlin_dt
```

**What changed:**
1. First, we localize the naive datetime as UTC using `UTC_TZ.localize()`
2. Then, we convert to Berlin timezone using `astimezone(BERLIN_TZ)`
3. This properly handles DST transitions
